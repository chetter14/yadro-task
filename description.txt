1. N / M временных лент длиной M:

	сортировка (M * log(M)) * (N / M) раз 
	проходимся по (N) элементам всего * (N / M) сравнений
	
	N * log(M) + N * N / M
	
2. 2 временных ленты длиной N:

	сортировка (M * log(M)) * (N / M) раз 
	локальный мерж по N (чуть меньше) элементам * (N / M) раз
	
	
Алгоритм работы:
	1) Создаём объект Tape для работы с входной лентой.
		Лента длиной N, хранит эл-ты integer типа.
		Внутри работа идёт с файлом.
		Внутреннее время задержки чтения/записи берётся из файла конфигурации.
	
	2) Создаём 2 внутренние ленты (2 объекта Tape класса) длиной N.
	3) Выполняем алгоритм сортировки по 2му пунтку выше.
	4) Из одной из внутренних лент выполняем копирование в выходную ленту.
		Копируем данные из одного объекта типа Tape в другой.
